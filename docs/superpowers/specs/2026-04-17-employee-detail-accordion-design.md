# 인원별 아코디언 상세 분석 — 설계 문서

**날짜:** 2026-04-17  
**프로젝트:** SKE-0004 신광전자 안전관리 시스템  
**대상 파일:** `admin.html` (제출현황 탭 — `renderCheckTab()`)

---

## 1. 목적

제출현황 탭에서 직원 이름을 클릭하면 해당 직원의 근골격계 데이터를 **인라인 아코디언**으로 펼쳐 보여준다. 시각적 그래프와 자동생성 개선권고로 관리자가 즉시 조치를 취할 수 있게 한다.

---

## 2. UX 결정 사항

| 항목 | 결정 |
|------|------|
| 레이아웃 | 행 인라인 아코디언 (이름 클릭 시 그 자리에서 펼침) |
| 동시 열림 | 한 명만 (다른 직원 클릭 시 이전 것 자동 닫힘) |
| 내부 구성 | 3열 카드형 |
| 개선권고 | 데이터 기반 자동생성 (NIOSH 판정 활용) |
| 확장성 | `expandedEmp` 단일값 → 배열로 바꾸면 다중 열림 전환 가능 |

---

## 3. 3열 카드 구성

### 1열 — 신체부위별 통증 가로바 그래프
- 부위: 목 / 어깨 / 팔·팔꿈치 / 손·손목·손가락 / 허리 / 다리·발
- 데이터 소스: `survey.painData[part].level`
- 바 너비 매핑:
  - `약한 통증` → 25% (초록)
  - `중간 통증` → 50% (노랑)
  - `심한 통증` → 75% (빨강)
  - `매우 심한 통증` → 100% (빨강)
  - 통증 없음 → 5% (회색)
- 통증 부위가 없으면 "통증 없음" 안내 표시

### 2열 — 종합 판정
- 종합 판정: `judgeOverall(survey)` 결과 표시
  - 정상 (초록) / 관리대상자 (노랑) / 통증호소자 (빨강)
- 유해요인 점수: `hazardData`에서 해당 직원 점수 표시 (없으면 "미작성")
- 통증 부위 수: `survey.painParts.length`

### 3열 — 개선권고 자동생성
기존 `judgePartNIOSH()` 로직을 활용하여 우선순위별 텍스트 생성:

| 등급 | 조건 | 색상 |
|------|------|------|
| 🚨 긴급 | 해당 부위 NIOSH 판정 = `통증호소자` | 빨강 |
| ⚠️ 권장 | 해당 부위 NIOSH 판정 = `관리대상자` | 노랑 |
| 💚 검토 | 통증 있으나 정상 판정 / 유해요인 점수 있음 | 초록 |

**자동생성 텍스트 규칙:**
```
통증호소자 부위 → "[부위] 전문의 검진 권장"
관리대상자 부위 → "[부위] 스트레칭 및 작업환경 개선 검토"
정상이나 통증 있음 → "[부위] 예방적 스트레칭 권고"
유해요인 점수 ≥ 70 → "작업환경 개선 우선 대상"
유해요인 점수 ≥ 50 → "작업방식 개선 검토 필요"
개선권고 없음 → "특이사항 없음 ✅"
```

---

## 4. 상태 관리

```javascript
var expandedEmp = null; // 현재 펼쳐진 직원 이름 (null = 모두 닫힘)
```

- 이름 클릭 → `toggleEmpDetail(name)` 호출
- 같은 이름이면 닫기 (`expandedEmp = null`)
- 다른 이름이면 교체 (`expandedEmp = name`)
- 이후 `renderCheckTab()` 재호출하여 반영

---

## 5. 데이터 없는 경우 처리

| 케이스 | 처리 |
|--------|------|
| 증상조사 미제출 | 아코디언 클릭 비활성 또는 "제출된 데이터 없음" 표시 |
| 유해요인 미작성 | 2열에 "유해요인 미작성" 회색 표시 |
| 통증 없음 (`hasPain !== '예'`) | 1열에 "통증 없음 ✅" 표시, 3열에 "특이사항 없음" |

## 6. 변경 범위

- **수정 파일:** `admin.html` 단일 파일
- **수정 함수:** `renderCheckTab()` — 직원 행 HTML 생성 부분
- **추가 변수:** `expandedEmp`
- **추가 함수:** `toggleEmpDetail(name)`, `buildDetailHTML(survey, hazard)`
- **기존 로직 재사용:** `judgePartNIOSH()`, `judgeOverall()`, `PART_ORDER`, `PART_ICON`

---

## 7. 확장 고려

- 다중 열림: `expandedEmp = null` → `expandedEmps = []` 로 변경 시 구현 가능
- 추후 인쇄: `buildDetailHTML()` 결과를 인쇄 모달에 재사용 가능
